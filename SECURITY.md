# Contributing

Thanks for your interest in contributing to AI Growth OS.

This project is practical by design. The goal is not to create abstract theory, but reusable systems that help people do better work.

## Good contributions

You can contribute:

- prompts that produce reliable outputs
- templates for briefs, audits, reports, and workflows
- examples from real business or creative contexts
- improvements to clarity and structure
- small scripts or automation ideas
- translations

## Contribution principles

Before submitting, check that your contribution is:

1. Clear  
2. Reusable  
3. Specific  
4. Practical  
5. Easy to adapt  

Avoid vague advice and generic AI output.

## File style

Use Markdown.

Keep documents structured:

```markdown
# Title

## Goal

## When to use

## Inputs

## Process

## Output

## Checklist
```

## Pull request checklist

- [ ] The contribution solves a clear problem
- [ ] The format is easy to reuse
- [ ] The language is simple
- [ ] There are no private client details
- [ ] The file is placed in the correct folder
