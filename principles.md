# Client Onboarding Workflow

## Goal

Move a new client from signed agreement to clear execution without chaos.

## Steps

### 1. Collect context

Ask for:

- business description
- offer
- audience
- current funnel
- traffic sources
- CRM access
- analytics access
- previous reports
- brand materials
- competitors
- deadlines

### 2. Create working folder

Recommended structure:

```text
/client-name
  /brief
  /analytics
  /ads
  /crm
  /content
  /reports
  /assets
```

### 3. Define roles

| Area | Owner | Backup |
|---|---|---|
| Strategy |  |  |
| Ads |  |  |
| CRM |  |  |
| Content |  |  |
| Analytics |  |  |
| Approvals |  |  |

### 4. Audit first

Do not start execution before checking:

- tracking
- offer
- landing page
- CRM stages
- lead response time
- content consistency
- sales process

### 5. Set first 14-day plan

The first two weeks should produce clarity:

- what works
- what is broken
- what to fix first
- what metrics matter
- who owns each task

## Output

A simple operating document with priorities, owners, and deadlines.
